# Usability Testing Checklist (Nielsen Heuristics)

## 1. Visibility of System Status
- [ ] Loading states shown when fetching data
- [ ] Success confirmation after create/update/delete
- [ ] Error messages explain what went wrong
- [ ] Streaming chat shows typing/generating indicator

## 2. Match Between System and Real World
- [ ] Labels use plain language (not API jargon)
- [ ] Icons are self-explanatory or have tooltips
- [ ] Terminology consistent throughout the app
- [ ] Agent/Crew/Skill terminology explained on first encounter

## 3. User Control and Freedom
- [ ] Cancel button available during multi-step flows
- [ ] Confirmation dialog before destructive actions
- [ ] Back navigation works as expected
- [ ] Form data preserved if page accidentally navigated away

## 4. Consistency and Standards
- [ ] Same actions trigger same results across pages
- [ ] Consistent button placement (primary action always same position)
- [ ] Same color for same status across all pages (green=active, red=error)

## 5. Error Prevention
- [ ] Required fields clearly marked
- [ ] Validation feedback before form submit
- [ ] Dangerous actions require confirmation

## 6. Recognition Rather Than Recall
- [ ] Available options visible without memorization
- [ ] Current page/section highlighted in navigation
- [ ] Recently used items accessible

## 7. Flexibility and Efficiency
- [ ] Common tasks achievable in 3 clicks or fewer
- [ ] Keyboard shortcuts for power users (where applicable)
- [ ] No unnecessary steps in main workflows

## 8. Aesthetic and Minimalist Design
- [ ] Pages not cluttered with rarely used information
- [ ] Most important action is visually prominent
- [ ] Data tables show most relevant columns by default

## 9. Help Users Recognize and Recover from Errors
- [ ] Error messages say what happened and how to fix it
- [ ] 404 / empty states guide users to next action
- [ ] Form validation highlights the specific field

## 10. Help and Documentation
- [ ] Tooltips on complex fields
- [ ] Skill descriptions explain what they do
- [ ] Empty states explain how to get started
