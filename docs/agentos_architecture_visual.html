Chat Request Data Flow
01
User Input
User sends message via web console chat interface (Next.js).
02
API Route
FastAPI receives POST to /conversations/{id}/chat or /chat/stream.
03
Agent Runtime
Orchestration layer resolves agent config, loads skills, injects tool context.
04
LLM + Tools
Agent calls LLM provider. Invokes MCP tools as needed. Supervisor may pause for approval.
05
Response
Result persisted to PostgreSQL. Streamed or returned to frontend. Redis updates cache.
